## Step-by-Step Onboarding Checklist
1. Create AD Account

Location: OU=<Department>,OU=Users,DC=servicedesk-simulator,DC=com (e.g., OU=Engineering,OU=Users,DC=servicedesk-simulator,DC=com)
• Username: firstname.lastname
• Email: firstname.lastname@servicedesk-simulator.com
• Set initial password (must meet policy: 12+ chars, mixed case, special char)
• User must change password on first login

2. Group Membership

• Add to "Domain Users" (default)
• Add to department group (Sales, Engineering, Finance, etc.)
• If manager/admin: Add to respective admin groups
• If remote: Add to "VPN-Access" group

3. Workstation Assignment

• Assign from inventory (naming: SD####)
• Configure: Workstation OS, Office Suite, required software
• Join to servicedesk-simulator.com domain
• Test login with new credentials

4. Mail Server Mailbox Creation

• Create mailbox on EXCH01 within 2 hours of AD account
• Set display name: FirstName LastName
• Set email policy: firstname.lastname@servicedesk-simulator.com
• Set mailbox size limit: 5GB quota
• Enable OWA and ActiveSync

5. Distribution Lists

• Add to all-company@servicedesk-simulator.com
• Add to department-specific list (sales-team, etc.)
• If manager: Add to managers list
• Verify via global address book

6. MFA Enrollment

• Send link: https://mfa.servicedesk-simulator.com/setup
• User must enroll before first VPN/remote access
• Issue backup codes (5), user stores securely
• Verify enrollment tested

7. Remote Worker Setup (if applicable)

• Ship SD Laptop Pro + docking station
• Pre-configure: VPN, WiFi, file shares
• Create VPN access account
• Ship with welcome letter + temporary password

8. Welcome Communication

• Send welcome email to user from manager
• Include: credentials, URL for password change, MFA link, IT helpdesk number
• Cc: manager for awareness
• Include first day orientation info


## Step-by-Step Offboarding Checklist

1. Disable AD Account (DO NOT DELETE)

• Location: Directory Users and Computers
• Right-click account > Disable Account
• Disabled users cannot login
• Note: Keep account for 1 year in case of rehire

2. Reset Password

• Set to random complex password (user won't know it)
• User cannot reset on own
• Account stays disabled, so password reset won't matter

3. Remove All Licenses

• Remove: Office Suite, any assigned software licenses
• Frees up licenses for reassignment to new employees
• License removal prevents access to cloud apps (Chat, Cloud Drive, etc.)

4. Remove from Security Groups

• Remove from: Department groups, admin groups, VPN, managers list
• Keep in: Domain Users (required for AD)
• Result: User cannot access shares, printers, or internal resources

5. Revoke MFA Access

• Cloud Identity > MFA > Remove user enrollment
• Delete any backup codes
• Prevents MFA re-enrollment

6. Email Auto-Reply

• Set in Mail Server: "This user is no longer with SDSim. Please contact [manager]."
• Subject line: "Out of Office"
• Duration: 30 days OR until mailbox transfer
• Test: Send test email to verify

7. Email Forwarding

• Forward all incoming mail to user's manager for 30 days
• Or: Forward to department distribution list
• Document in ticket: Where mail forwarded to
• Auto-remove forwarding after 30 days via reminder

8. Convert to Shared Mailbox (After 30 Days)

• Disable auto-reply
• Keep mailbox live (search-able for compliance)
• Convert to shared mailbox for manager access
• Grant manager full access

9. Collect Hardware Assets

• Retrieve laptop, monitor, keyboard, mouse, docking station
• Return to IT inventory
• Check out device tag numbers in asset system
• Wipe drive if reissuing to new user

10. Document in Ticket

• Record date disabled
• List of groups removed from
• Hardware collected
• Email forwarding destination
• Manager notification date
• Keep ticket for compliance


## Installation Process

1. Check Approved Software List

• Is software on approved list? (Office Suite, Video Meet, Team Messenger, etc.)
• If not: Reject request, inform user to contact manager + IT leadership
• If yes: Proceed to license check

2. Verify License Availability

• Check License Inventory (see Software & Licensing article)
• Is license seat available (not all used)?
• If no seats: Reject, escalate to Tier 2 for license purchase
• If yes: Proceed to installation

3. Imaging Server-Managed Applications (Recommended)

• Direct user to selfservice.servicedesk-simulator.com
• User logs in with domain account
• User clicks "Available Software"
• User selects app to install (e.g., PDF Reader Pro, Video Meet)
• App installs automatically via Imaging Server agent
• Completion: 10-15 minutes
• License auto-assigned to user account

4. Manual Installation (Non-Imaging Server Apps)

• Remote into user's machine (helpdesk-install account)
• Download installer from trusted source (publisher website, not generic)
• Run as Administrator
• Follow install wizard
• Decline bloatware/extra software
• Test app launches successfully
• Document license assignment in ticket

5. License Assignment

• For Office Suite: User auto-licensed at account creation
• For other apps: Record in spreadsheet or licensing portal
• Update License Inventory tracker
• Notify user: "License assigned for 1 year, renewal in [date]"

6. Verification

• User confirms app launches
• Features available (not trial or limited version)
• Shortcut on desktop or Start menu
• Close ticket with license assignment details


## Adding Network Printer to Workstation OS

1. Open Settings

• Right-click Start menu > Settings
• Or: Click gear icon in taskbar

2. Navigate to Printers & Scanners

• Settings > Devices > Printers & Scanners
• Click "Add a printer or scanner"
• Wait 10 seconds for network scan

3. Browse & Select Printer

• If printer appears in list: Click > Add printer
• If not: Click "Browse all printers"
• Expand PRINT01 network path
• Locate printer by floor (Floor1-Printer-01, Floor2-Printer-02, etc.)
• Click > Select

4. Install Driver (if needed)

• If prompted for driver: Click "Universal Print Driver"
• If driver not available: Download from the printer vendor support site
• Select printer model (e.g., office color laser MFP)
• Complete installation

5. Print Test Page

• Printer added to Settings > Devices > Printers & Scanners
• Right-click printer > Printer properties
• Devices and Printers tab > Print a test page
• Verify page prints from selected printer

6. Set as Default (Optional)

• Right-click printer > Set as default
• Applies to all new print jobs (Ctrl+P dialog)
• User can override per-job in print dialog
Troubleshooting
Printer Not Found:
PRINT01 offline, network issue, try manual IP entry (10.0.2.5x)
Driver Installation Fails:
Restart computer, download latest printer driver
Test Page Fails:
Check printer power, network cable, IP address reachability (ping 10.0.2.50)
